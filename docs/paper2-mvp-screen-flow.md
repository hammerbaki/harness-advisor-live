# Paper 2 — MVP 화면 흐름 (초안)

> 상태: draft. 목적 = "채팅"이 아니라 **리서치 워크플로**로서의 MVP 화면 흐름을 정의한다.
> 핵심 흐름: **질문 → 근거 패널 → 답변(인용) → 저장된 투자 노트 → 공유 가능한 리서치 문서.**
> 현재 UI는 모바일 브리핑(`src/main.tsx`); 이를 데스크톱 리서치 워크스페이스로 확장한다.

## 1. 핵심 사용자 여정

```
[진입/로그인] → [워크스페이스]
        질문 입력
          │
          ▼
   근거 검색(harness: DART/KRX/뉴스 + 클레임)
          │
          ▼
   ┌─────────────┬──────────────────────┐
   │  답변(인용) │  근거 패널(출처/표/차트) │
   └─────────────┴──────────────────────┘
          │
          ├─→ [노트에 저장]  (블록/마크다운/태그/티커)
          │
          └─→ [리서치 문서로 편집] → [공유]
                                        │
                                        ▼
                               공유 문서에서 다시 질문(루프)
```

## 2. 데스크톱 워크스페이스 레이아웃 (와이어프레임)

```
┌───────────────────────────────────────────────────────────────┐
│ Topbar: 그룹/종목 선택 · 검색 · [정보 제공, 투자 조언 아님] 고지  │
├──────────────┬───────────────────────────────┬────────────────┤
│  대화(좌)     │  근거 패널(중)                 │  노트/문서(우)  │
│              │                               │                │
│  Q1 …        │  · 출처 카드 (제목/발행/링크)   │  [현재 노트]    │
│  A1 (인용¹²)  │  · 재무 표 / 시계열 차트        │  - 저장한 답변  │
│  Q2 …        │  · 공시 핵심 변화 하이라이트    │  - 태그/티커    │
│  [입력창]     │  · 인용 클릭 → 원문 위치        │  [문서로 편집]  │
│              │                               │  [공유]        │
├──────────────┴───────────────────────────────┴────────────────┤
│ Footer: 상시 고지 · 데이터 신선도/모드(live/fixture) 배지        │
└───────────────────────────────────────────────────────────────┘
```

- **좌(대화):** 멀티턴 질문·답변. 답변의 모든 클레임에 인용 번호. (신규)
- **중(근거 패널):** 출처 카드·표·차트, 인용 클릭 시 원문 위치로. 논문 1의 클레임-소스
  링크를 그대로 표면화. (일부 재사용)
- **우(노트/문서):** 답변을 블록으로 저장, 태그·티커 부여, 리서치 문서로 편집·공유. (신규)

## 3. 화면·상태 목록

| # | 화면/상태 | 설명 | 재사용/신규 |
|---|---|---|---|
| S0 | 진입·로그인 | (연구용은 로그인 생략/간이 세션 가능) | 신규(간이) |
| S1 | 빈 워크스페이스 | 예시 질문 프롬프트 | 신규 |
| S2 | 질문 처리 중 | 근거 검색 로딩·모드 배지 | 재사용(harness) |
| S3 | 답변+근거 | 인용 답변 + 근거 패널 | 확장 |
| S4 | 노트 저장 | 블록 저장·태그·티커 | 신규 |
| S5 | 문서 편집 | 여러 노트 → 리서치 문서 | 신규 |
| S6 | 공유 | 읽기 전용 공유(+고지) | 신규 |
| S7 | 오류/폴백 | 소스 실패 시 폴백 상태 표기 | 재사용(harness) |

## 4. 계약 준수 (필수)

- 모든 신규 표면(멀티턴 답변·노트·공유 문서)은 `server/index.mjs`의
  leakage/link/language 검사와 `validateStructuredAdvisorOutput` 게이트를 통과.
- 공유 문서·노트에도 상시 비자문 고지. 인용 없는 클레임 노출 금지.
- 상세: [안전·범위 정책](paper2-safety-scope-policy.md).

## 5. MVP 절취선 (무엇을 빼는가)
- 미국 시장·유료 데이터·자동매매·포트폴리오 연동 → **제외.**
- 실서비스 인증/결제 → 연구용 간이 세션으로 대체.
- 목표: 사용성 평가 태스크 5개를 수행 가능한 **최소 워크플로**.
