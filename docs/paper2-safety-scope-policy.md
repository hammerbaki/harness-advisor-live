# Paper 2 — 안전·범위 정책 (초안)

> 상태: draft. 목적 = 제품을 **"출처 기반 투자정보 이해 지원 에이전트"**로 범위 고정하고,
> "개인화 투자 조언"과의 경계를 코드로 강제할 수 있는 형태로 명문화한다.
> 원칙: 논문 1 harness의 계약(leakage/link/language)을 **완화하지 않고 오히려 유지·강화**한다.

## 1. 한 문장 정의

> 공개 출처에 근거해 투자 관련 **정보의 이해**를 돕는 에이전트. 매수/매도/보유/목표가/비중/타이밍
> 등 **개인화된 투자 판단은 산출하지 않는다.** (Not investment advice.)

## 2. 범위 (In-scope vs Out-of-scope)

| In-scope (허용) | Out-of-scope (차단) |
|---|---|
| 사실·수치의 출처 링크 제공 | 매수/매도/보유(buy/sell/hold) 권유 |
| 재무·공시·시세 신호의 **설명** | 목표주가·적정주가 제시 |
| 리스크·모니터링 포인트 정리 | 포지션 비중·자산 배분 추천 |
| 종목·기간·항목 **비교** | 진입/청산 타이밍 조언 |
| 후속 질문·체크리스트 생성 | 개인 상황 기반 맞춤 조언 |
| "왜 그런가"의 근거 추적 | 수익률 보장·예측 단정 |

## 3. 경계 판정 규칙 (edge cases)

- **정보 vs 조언:** "삼성전자 영업이익이 왜 늘었나"(허용) vs "삼성전자 지금 사야 하나"(차단).
- **비교 vs 추천:** "A와 B의 밸류에이션 배수 비교"(허용) vs "A가 B보다 낫다"(차단).
- **리스크 정리 vs 경고성 조언:** "부채비율 상승 항목 나열"(허용) vs "위험하니 피하라"(차단).
- **미래 언급:** 출처가 명시한 가이던스 인용은 허용, 모델의 독자적 예측·단정은 차단.

## 4. 강제 지점 (프롬프트가 아니라 코드)

논문 1의 3대 검증 계열을 **그대로 재사용·유지**한다:

1. **Language family** — 매수/매도/목표가 등 추천 표현 차단 (`visibleAnswer` 언어 검사).
   → Paper 2에서 대화형·노트로 표면이 늘어나므로 **동일 검사를 모든 신규 표면에 적용**.
2. **Link family** — 모든 인용은 소스 패키지/문서화된 폴백 상태로 resolve.
   → 노트·공유 문서의 인용도 동일 계약 적용.
3. **Leakage family** — 내부 클레임 ID·트레이스·프롬프트/eval 어휘·JSON 유출 차단.
   → 대화 메모리·공유 문서에서 내부 식별자 유출 방지.

추가로 `validateStructuredAdvisorOutput` + 결정론적 폴백 게이트를 **모든 신규 출력 경로
(멀티턴 응답, 저장 노트, 공유 문서)에 통과**시킨다. 프롬프트에 사실·정책을 넣지 않는
논문 1의 분리 원칙을 유지한다.

## 5. 규제·고지 (한국 맥락)

- **자본시장법:** 투자자문업/유사투자자문업 미등록 상태에서 개인화 투자권유 금지.
  본 제품은 **정보 이해 지원**으로 한정하여 자문 행위에 해당하지 않도록 설계.
- **사용자 고지:** 모든 답변·노트·공유 문서에 "정보 제공이며 투자 조언이 아님, 투자 판단과
  책임은 사용자에게 있음" 상시 고지.
- **실험 맥락(사용성 평가):** 피험자가 실제 매매를 하도록 유도하지 않음. 모의·이해 과제로 한정.

## 6. 미확정 결정 (기획 확정 필요)
- 대화형에서 사용자가 노골적으로 조언을 요구할 때의 **거부/재구성(refusal-reframe) 문구** 표준화.
- 공유 문서의 공개 시 추가 고지·워터마크 필요 여부.
- Throne을 실험 자극물로 쓸 경우의 경쟁사 ToS·저작권 검토(별도).
