# Paper 2 — 사용성 평가 태스크 (초안, 5개)

> 상태: draft. 목적 = 제품 기능과 짝을 이루는 **사용성 평가 태스크 5개**를 정의한다.
> 각 태스크는 (a) 시나리오, (b) 측정 대상, (c) 성공 기준, (d) harness 속성과의 연결을 갖는다.
> 연구 초점: **신뢰 보정(trust calibration)** 과 **적정 의존(appropriate reliance)** —
> 코드로 강제된 출처 계약·비추천이 사용자 이해·검증 행동에 미치는 영향.

## 공통 측정 지표 (per task)
- **과업 성공(task success)**: 이진/부분 점수.
- **소요 시간(time-on-task)**, **오류 수**, **도움 요청 수**.
- **SUS**(세션 종료 후 전체), **신뢰 척도**(과업별 사전/사후).
- **적정 의존**: 정답/오정보 상황에서 사용자가 출처로 **검증**하는 비율.
- **think-aloud** 정성 코딩(선택).

## 태스크 1 — "왜 주가가 움직였나"
- **시나리오:** 특정 종목의 최근 실적/이벤트를 근거로 변동 원인을 설명하게 한다.
- **측정:** 근거를 **출처와 연결**해 설명하는가; 모델의 단정 vs 출처 기반 구분.
- **성공 기준:** 최소 N개의 출처-연결 원인 식별, 인용 확인 1회 이상.
- **harness 연결:** link/leakage — 인용 resolve·내부식별자 비노출.

## 태스크 2 — "최근 공시 핵심 변화 찾기"
- **시나리오:** 최근 DART 공시에서 투자자가 주목할 변화를 찾게 한다.
- **측정:** 원문 대비 핵심 변화 포착 정확도; 원문 위치로의 **인용 추적** 사용.
- **성공 기준:** 핵심 변화 항목 식별 + 원문 근거 확인.
- **harness 연결:** link — 인용 클릭→원문 위치.

## 태스크 3 — "두 종목 비교"
- **시나리오:** 두 종목/그룹을 지정 항목(밸류에이션·재무 추세 등)으로 비교하게 한다.
- **측정:** 비교가 **추천으로 미끄러지지 않는지**(사용자 인식); 비교표 이해도.
- **성공 기준:** 항목별 비교 완료, "어느 쪽을 사라"류 결론 유도 시 시스템이 **비추천 유지**.
- **harness 연결:** language — 추천어 차단(경계 판정 규칙 검증).

## 태스크 4 — "출처 검증하기"
- **시나리오:** 제시된 답변의 특정 클레임이 실제 출처에 부합하는지 확인하게 한다.
  (일부 세션에 **의도적 오정보/약한 근거**를 삽입해 적정 의존 측정.)
- **측정:** 사용자가 출처로 검증하는가; **오정보를 잡아내는 비율**(적정 의존 핵심 지표).
- **성공 기준:** 오정보/약근거 클레임을 출처 대조로 식별.
- **harness 연결:** link/leakage — 모든 클레임의 검증 가능성.
- **주의:** 오정보 삽입은 **연구 조작(manipulation)**으로만; 실제 배포엔 없음. IRB 고지.

## 태스크 5 — "리스크 체크리스트 만들기"
- **시나리오:** 특정 종목의 리스크·모니터링 포인트를 체크리스트로 정리하고 노트에 저장하게 한다.
- **측정:** 워크플로(답변→노트→문서) 완주율; 노트/문서의 인용 보존.
- **성공 기준:** 출처-연결 리스크 항목으로 체크리스트 저장, 공유 문서 생성.
- **harness 연결:** 전 계열 — 노트·공유 문서까지 계약 유지.

## 조건(condition) 설계 후보
- **A: 우리 시스템(harness, 출처 계약 강제)**
- **B: prompt-only 베이스라인**(논문 1의 ablation) 또는 **Throne류 상업 베이스라인**(자극물)
- within/between-subjects, counterbalancing, 순서 효과 통제 — 확정 필요.

## 미확정 결정
- formative(n≈5–8) vs summative(n≈20–40+) → 지표·통계 확정.
- 신뢰 척도 도구 선택(예: 기존 trust-in-automation 척도 번안).
- 태스크 4의 오정보 삽입 범위·윤리 고지(IRB).
- 피험자 프로파일(리테일 투자자, 한국어 금융 리터러시 수준).
