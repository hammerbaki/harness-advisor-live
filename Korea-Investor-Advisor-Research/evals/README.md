# Evaluations

Evaluation suites should be split into:

- offline deterministic tests using fixtures
- live tests requiring external credentials
- paper scenarios with frozen expected validator outcomes

Each scenario should record:

- group ID
- user question
- source inputs
- tool calls
- model/prompt/wiki versions
- final answer
- validator results

## Frozen Scenarios

`scenarios/` contains paper-oriented evaluation scenario definitions. These are
not generated trace outputs; they are the frozen questions, expected claim IDs,
trace expectations, and answer-level signals used to evaluate whether a target
group is ready to become a paper/demo slice.

Run:

```bash
npm run validate:evals
```

Run the current advisor auto-evaluation baseline:

```bash
npm run eval:advisor
```

The auto-eval runner adapts an autoresearch-style loop to this finance advisor:
fixed scenario, fixed rubric, API run, score, and keep/discard decision. It
uses `evals/rubrics/advisor.answer-quality.v0.2.json` by default.

Rubric v0.2 adds customer-facing briefing quality checks adapted from the
original PoC's briefing-card rules: no vague adjectives, no numeric
paraphrase-only answer, a clear judgment axis, prioritized monitoring points,
no buy/sell/target-price recommendation language, and no excessive reuse of
one generic table of contents when the question intent changes.

The first frozen scenario set is `hanwha.reference-slice.json`. Samsung, SK,
Hyundai Motor, and LG should receive equivalent files only after their source
manifests and source-backed claim sets exist.

## Result Summaries

`results/` contains paper-safe summaries of frozen scenario runs. These files
summarize runtime modes, required claim coverage, and paper/table suitability
without publishing raw client or operational logs.

The first auto-eval baseline is:

```text
evals/results/hanwha-reference-slice-v0.1.autoeval-baseline.2026-05-02.json
```

## Trace Exports

`traces/` contains local JSON exports generated by the advisor API for paper
and demo evaluation. Each file preserves the answer, trace envelope,
source-backed claims, links, and tool status for a single run.

These traces are local evaluation artifacts. Review them before sharing because
future live runs may include client-specific questions or operational metadata.
